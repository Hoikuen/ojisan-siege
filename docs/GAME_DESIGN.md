# おじさん襲来（OJISAN SIEGE）— GAME DESIGN

## コンセプト
チビおじさん軍団がわらわら襲来するのを、タワーを配置して防ぐタワーディフェンス（Kingdom Rush風）。
「敵が大量に可愛く来る気持ちよさ」×「配置の戦略」。

## 最初のゴール（vertical slice・図形プレースホルダーで）
絵はまだ描かず、次が成立する最小ループを完成させる：
- 1マップ（道を1本）。敵が列で進軍してくる
- 道の脇にタワーを配置できる（2種類）。敵を自動で攻撃
- 敵を倒すとお金 → タワーを強化／増設
- 3ウェーブを耐えきる → クリア／ライフ0で敗北
- リトライできる
これが「面白い」と確認できてから、絵を作り込んで差し替える。

## コアループ
敵が経路を進む → タワーで迎撃 → お金で強化・増設 → ウェーブを耐える → クリア → 次マップ

## 要素（絵は後で差し替え）
- 敵：小さいおじさん・おじさん系の芋虫・動物。ボス＝でかいおじさん
- タワー（おじさんが運用する防衛施設）：警備員おじさん・立ち食いそば屋台 等。強化可能
- 可愛くわらわら来る敵を、的確な配置で止めるのが快感

## アート方向性（差し替え時）
斜め見下ろしの箱庭マップ。ぷっくりデフォルメ。敵はチビで大量に可愛い。タワーのおじさんも表情豊か。

## 技術メモ
敵は決められた経路を進むだけ＋設置タワー＋弾。プラットフォーマーの物理と違い当たり判定が単純で安全。
難所はバランス（敵の強さ・お金の出方・ウェーブ設計）→最小ループで先に詰める。
Phaser 3 + Vite を想定。詳細は ~/Developer/games/PHASER3_CONSTRAINTS.md。
