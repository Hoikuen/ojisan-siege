## 現在の状態

**最小ループ（vertical slice）完成・図形プレースホルダー段階**。絵はまだ描かない方針（実装が固まってから差し替え）。

- エンジン：Phaser 3.90 + Vite 5
- 物理エンジンは不使用。敵は経路上の「進んだ距離(dist)」だけ持ち、毎フレーム経路を補間して座標化する。
  タワーは距離判定で索敵、弾は対象へホーミング。→ `PHASER3_CONSTRAINTS.md` の物理（body駆動・床めり込み）の落とし穴を構造的に回避。

## 実装済み機能

- 2マップ（住宅街3レーン／商店街4レーン）。`src/data/maps.js` にデータで定義（足すだけで増える）
- 敵4種：チビおじ(grunt)／いそぎおじ(runner)／ごりおじ(brute)／ボスおじ(boss)
- タワー2種：警備員（単体）／立ち食いそば屋台（着弾範囲）。各3段階強化・売却（投資の60%払い戻し）
- 経済：撃破報酬＋ウェーブ突破ボーナス。所持金で増設・強化
- 各マップ3ウェーブ。全ウェーブ耐えるとマップクリア→次マップ。全マップ制覇でゲームクリア。ライフ0で敗北・リトライ
- 効果音（Web Audioの手続き合成・アセット不要）＋ミュート。ゲーム速度1x/2x・一時停止
- UI：建設/強化/売却メニュー、射程プレビュー、HUD（金/ライフ/マップ・ウェーブ）

## 起動・確認

- ローカル起動：`npm run dev`（このマシンでは他セッションが5173を使うことがあるため、衝突時は `npx vite --port 5199 --strictPort`）
- 本番ビルド：`npm run build`（Vite devは遅延評価でimport漏れを見逃すため、必ずbuildも通す）
- localhost / 127.0.0.1 は共有用URLとして案内しない

## 設計判断のメモ

- 「データ（content.js / maps.js）」と「エンジン（GameScene）」を分離（starter-kit RESKIN思想）。絵もデータも差し替えで別ゲーム化できる形を維持
- マップ固有値（経路・スロット・ウェーブ・初期金/ライフ）は `maps.js` に一本化。`config.js` は画面サイズ・色のみ（二重管理を避ける）
- 検証用に `window.__game` を公開（本番でも無害）
